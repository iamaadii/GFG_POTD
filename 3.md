## Problem Statement
```
Given an integer n, return all the n digit numbers in increasing order, such that their digits are in strictly increasing order(from left to right).
```

### Constraints:
```
1 ≤ n ≤ 10^5
```


## Approach-1:
- Initialize an empty list to store the increasing numbers. 
- If N > 9, return the empty list since a strictly increasing number cannot have more than 9 digits. 
- If N == 1, add 0 to the result. 
- Start a DFS/Backtracking process from each digit 1 to 9. 
- At each step:
  - If the current length becomes N, add the formed number to the result. 
  - Otherwise, try all digits greater than the previously chosen digit (prev + 1 to 9). 
  - Append the digit, recurse, and then backtrack.
- Return the list of generated numbers.

---

### Code:
```java
class Solution {
    
    static ArrayList<Integer> arr;
    static int n;
    
    static void helper(int index,int prev,int num){
        
        if(index==n){
            arr.add(num);
            return;
        }
        
        for(int digit=prev+1;digit<=9;digit++){
            helper(index+1,digit,num*10+digit);
        }
    }
    
    public static ArrayList<Integer> increasingNumbers(int N) {
        // code here
        arr = new ArrayList<>();
        n=N;
        
        if(n>9) return arr;
        
        if(n==1) arr.add(0);
        
        for(int start=1;start<=9;start++){
            int num = start;
            helper(1,start,num);
        }
        return arr;
    }
}

```

### Complexity
```
Time Complexity = O(N × C(9, N)) because we generate all increasing numbers of length N.

Since digits are restricted to 1–9, this is bounded by a constant, so practically O(1).
```

```
Space Complexity = O(N) recursion stack (excluding the output array).
```
